# GoodReason Systemic Method

**GoodReason Systemic Method** is a SOI-centred, SuperJSON-guided and AI-compatible epistemic modelling method for systems science.

It places a **System of Interest (SOI)** at the centre of inquiry and organises systemic knowledge through eight α–Ω perspectives and seven depth circles. Its purpose is to transform fragmented systems knowledge into a visual, symbolic, computational and epistemological architecture that can be analysed, queried, validated, specialised and developed.

This repository contains the compact method kernel.

## Core claim

GoodReason treats systems science as an epistemic modelling problem.

A source material, case, theory, standard, research field or practical challenge is interpreted as a System of Interest. The best GoodReason configuration is the α–Ω model that minimises the weighted residual error between the source material and the SuperJSON reference architecture.

```text
Source material → SOI → α–Ω mapping → circle 1–7 assignment
                → SuperJSON comparison → best-fit systemic model
                → solution, validation, feedback and further development
```

## Why this matters

Systems science contains a rich diversity of theories, methods, schools and traditions. The challenge is not only to list them, but to organise how systemic knowledge is formed, compared, validated and developed.

GoodReason provides a method for this by combining:

- systems science
- metascience
- epistemology
- cybernetics
- symbolic modelling
- visual reasoning
- JSON-compatible knowledge structures
- AI-assisted synthesis and validation

## The α–Ω perspectives

| Symbol | Perspective | Methodological role |
|---|---|---|
| α | Purpose / mind | Why the SOI is selected, what the inquiry serves, and what direction gives meaning to the whole. |
| π | Theory / principle | The axioms, concepts, arguments, theories, principles and criteria used to understand the SOI. |
| χ | Information / environment | Input, source material, context, isomorphism, data, signals, logic, mathematics and environmental relations. |
| ΔΨ | Renewal / pressure | Disturbance, contradiction, anomaly, disruption, failure, collapse risk, paradigm pressure and development demand. |
| β | Organisation / structure | Roles, validation systems, governance, coordination, viability, control and whole-system organisation. |
| φ | Design / solution | Output, intervention, solution logic, design value, system architecture and improvement over the input state. |
| τ | Integration / implementation | Implementation, interoperability, practice, technology, Science 2.0, AI support and knowledge integration. |
| Ω | Developmental arc / feedback | Feedback, learning, recognition, legitimacy, disciplinary authority, resilience and long-term development. |

## Suggested AI use

The model can be used as a structured prompt/configuration layer for AI-assisted systems inquiry.

Example instruction:

```text
Analyse the given source material as a System of Interest.
Map the material to the GoodReason α–Ω perspectives and circles 1–7.
Compare alternative mappings against the SuperGoodReasonModel.
Report the best-fit configuration, residual mismatches and missing systemic functions.
```
