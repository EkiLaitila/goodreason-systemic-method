# GoodReason Systemic Method

## 1. Definition

**GoodReason Systemic Method** is a SOI-centred, SuperJSON-guided and AI-compatible method for systems science.

It defines systemic inquiry as a structured relation between:

1. a **System of Interest (SOI)**,
2. an explicit **α–Ω epistemic architecture**,
3. seven **depth circles**,
4. a universal **SuperJSON reference model**,
5. optional domain-specific **SpecialJSON models**,
6. a best-fit modelling process that selects the most coherent configuration.

The method is designed for systems science, systems engineering, metascience, AI-assisted synthesis, epistemic modelling and transdisciplinary learning.

## 2. The System of Interest at the centre

The SOI is placed at the centre of the model.

The key question is:

> What is being studied, interpreted, designed, improved or transformed?

The SOI may be:

- a practical system
- a scientific field
- a theory or method
- a text corpus
- a social organisation
- a technical architecture
- a standards framework
- an AI-supported knowledge process
- systems science itself

GoodReason does not begin with a school or discipline. It begins with the SOI and asks how the SOI is best understood through the α–Ω perspectives.

## 3. The α–Ω architecture

The eight GoodReason perspectives form the main epistemic coordinate system.

| Symbol | Role | Main question |
|---|---|---|
| α | Purpose / mind | Why does this inquiry matter? |
| π | Theory / principle | What concepts, axioms and principles make the SOI intelligible? |
| χ | Information / environment | What input, context, data, isomorphisms and environmental signals define the SOI? |
| ΔΨ | Renewal / pressure | What contradiction, anomaly, disruption or development pressure drives change? |
| β | Organisation / structure | How is the SOI organised, validated and governed? |
| φ | Design / solution | What output, solution, intervention or design value is produced? |
| τ | Integration / implementation | How is the solution integrated into practice, technology and knowledge work? |
| Ω | Developmental arc / feedback | What feedback, learning, legitimacy and long-term development result? |

This architecture is cybernetic: χ provides input, φ provides output, Ω provides feedback, ΔΨ provides change pressure, and τ integrates the result into practice.

## 4. Seven depth circles

Each α–Ω perspective has seven levels. These levels provide depth, maturity and consciousness coordinates for systemic inquiry.

The circles represent a movement from simple recognition and stable practice toward complexity, independent agency, emergence, disruption, philosophical reflection and collective orientation.

The circles make it possible to ask:

- Is this interpretation only descriptive?
- Is it theoretically grounded?
- Does it identify anomalies or development pressure?
- Does it produce a solution?
- Does it support implementation?
- Does it create feedback, learning and institutional development?
- Does it reach philosophical, ethical or metascientific reflection?

## 5. SuperJSON and SpecialJSON

`SuperGoodReasonModel.json` is the universal reference architecture.

It contains:

- model metadata
- eight sectors: α, π, χ, Δψ, β, φ, τ and Ω
- seven entries per sector
- cross-perspective descriptions for each entry
- seven depth circles
- methodological notes for AI-assisted and systemic use

A **SpecialJSON** model is a domain-specific refinement of the SuperJSON model.

Examples:

```text
SuperJSON
  → SpecialJSON for living systems
  → SpecialJSON for physical systems
  → SpecialJSON for sociotechnical systems
  → SpecialJSON for systems engineering
  → SpecialJSON for AI-supported science
  → SpecialJSON for governance
  → SpecialJSON for metascience
```

The universal model gives comparability. The specialised model gives domain precision.

## 6. Cybernetic input/output logic

A coherent GoodReason model should contain the following systemic functions:

```text
χ input
  → α orientation
  → π principles
  → ΔΨ pressure
  → β organisation
  → φ solution
  → τ implementation
  → Ω feedback
```

The order may vary in practical inquiry, but a strong model makes all functions visible.

A weak model often lacks:

- χ: reliable input and isomorphism
- φ: explicit output and design value
- Ω: feedback and learning
- ΔΨ: change pressure or anomaly
- β: validation and organisation
- τ: implementation route

## 7. Best-fit modelling

The preferred model is the configuration that gives the smallest residual mismatch between the source material and the SuperJSON reference architecture.

The residual mismatch includes:

- semantic error
- structural error
- missing α–Ω functions
- weak input/output logic
- unclear circle assignment
- unmodelled anomalies
- poor coherence between SOI, method and outcome

This turns AI-supported interpretation into a guided epistemic modelling process rather than free association.

## 8. AI-assisted use

GoodReason can support AI work by giving the model a stable reference structure.

Typical AI tasks:

- classify source material into α–Ω perspectives
- identify missing systemic functions
- compare alternative mappings
- generate SpecialJSON refinements
- produce diagrams, summaries and argument maps
- detect conceptual gaps
- reduce hallucination by requiring explicit fit and residual-error reporting

## 9. Relation to systems science and Science 2.0

GoodReason supports the modernisation of systems science by making systemic knowledge:

- visible
- symbolic
- computational
- queryable
- teachable
- comparable
- extendable
- AI-compatible

It also supports Science 2.0 by treating knowledge development itself as a system: reflexive, model-based, collaborative, computational and open to feedback.
